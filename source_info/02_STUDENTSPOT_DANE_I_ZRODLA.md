# StudentSpot — dane startowe i źródła

## 1. Zasada wiarygodności danych

Dane dzielimy na trzy poziomy:

- **potwierdzone** — pochodzą z oficjalnych stron AHE lub wskazanych serwisów sal konferencyjnych,
- **robocze** — przekazane przez autora projektu i wymagają potwierdzenia przed wdrożeniem produkcyjnym,
- **nieznane** — nie wolno ich uzupełniać na podstawie domysłów.

W aplikacji dane robocze mogą być użyte jako seed demonstracyjny, ale powinny mieć flagę `verification_status = "unverified"`.

## 7. Logotypy i branding

### Oficjalne materiały

https://www.ahe.lodz.pl/akademia/dla-mediow/do-pobrania

Pobrać:

1. pierwsze logo AHE w wersji polskiej,
2. drugie logo AHE w wersji angielskiej,
3. logo Samorządu Studentów,
4. logo Uniwersytetu Trzeciego Wieku.

Zasady dla agenta:

- pobrać pliki do `app/static/media/brand/`,
- zachować oryginalne proporcje,
- nie zmieniać kolorów,
- zapisać URL i datę pobrania w `assets_manifest.json`,
- nie hotlinkować obrazów w wersji finalnej,
- jeżeli pobranie nie jest możliwe, użyć neutralnego placeholdera tekstowego i pozostawić TODO.

## 12. Seed demonstracyjny

Dla niepotwierdzonych danych użyć:

```json
{
  "source_type": "user_provided",
  "verification_status": "unverified",
  "last_verified_at": null
}
```

Dla danych potwierdzonych:

```json
{
  "source_type": "official_or_listed_source",
  "verification_status": "verified",
  "source_url": "https://...",
  "last_verified_at": "2026-06-14"
}
```
